#  11. Брокеры сообщений	

## Реализация по задаче:
Разделил два сервиса на два репозитория, [второй репозиторий B.](https://github.com/kulickk/service-b-rabbitmq-task) 

### Конфигурация
- **application.yml**  
  Задает параметры подключения к RabbitMQ.

- **RabbitConfig**  
  Конфигурационный класс, создающий очередь, exchange и биндинги с использованием DirectExchange, а также настраивающий JSON-конвертер.

### Модель
- **ChatMessage**  
  Класс, представляющий JSON-сообщение.

### Сервис
- **SenderService**  
  Сервис, отправляющий сообщения в другую очередь через RabbitTemplate. Сообщения логируются.

### Обработчик
- **Reciever**  
  Компонент, помеченный @RabbitListener, получающий JSON-сообщения из очереди и выводящий их в логи.


### Контроллер
- **ControllerA**  
  REST-контроллер с POST эндпойнтом /api/a/send, который принимает тело сообщения и инициирует отправку через SenderService.
