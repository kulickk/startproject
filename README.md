#  Web Application Context Task-4

## WebApplicationContext
`WebApplicationContext` - интерфейс, для конфигурации веб-приложения. Расширяет базовый функционал ApplicationContext и добавляет возможность обработки HTTP-запросов, поддержку работы с `ServletContext`, поддержку скоупов и интеграцию с MVC. Контекст создается в методе `SpringApplication.run()`.

## Маппинги запросов

| Аннотация      | Описание |
|----------------|----------|
| `@GetMapping`  | Обрабатывает GET запросы, получение данных с сервера без изменений |
| `@PostMuping` | Обрабатывает POST запросы, создание новых данных на сервере |
| `@PutMapping`  | Обрабатывает PUT запросы, создание данных на сервере. Отличие от POST: множественные вызовы с идентичным набором данных дают тот же результат |
| `@PatchMapping`| Обрабатывает PATCH запросы, удаление данных с сервера |
| `@DeleteMapping`| Обрабатывает DELETE запросы, альтернатива PUT, когда не нужно отправлять весь объект |

## Параметры маппингов

| Параметр   | Описание |
|------------|----------|
| `value`, `path` | URL-путь |
| `method` | HTTP-метод (для `@RequestMapping`) |
| `params` | Фильтрация по параметрам запроса |
| `headers` | Фильтрация по HTTP-заголовкам |
| `consumes` | Ожидаемый Content-Type |
| `produces` | Возвращаемый Content-Type |

## Реализации по задаче:

### HeaderController
- По пути `/headers` на GET-запрос возвращает шаблон со списком всех заголовков запроса
- По пути `/headers/json` возвращает JSON с заголовками (без шаблона)

### JsonController
- Принимает JSON на вход (путь `/json`)
- Возвращает JSON с добавленным полем `id`

### Обработка ошибок
- `CustomException` - кастомное исключение для 502 ошибки
- `CustomExceptionHandler` - обработчик кастомного исключения
